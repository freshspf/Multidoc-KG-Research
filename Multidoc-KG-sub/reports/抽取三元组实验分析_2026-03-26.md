# Extraction 实验分析（2026-03-26）

---

## 1. 实验目的

当前 Extraction 实验的目标是：

- 验证 `agents/extraction.py` 是否已经从旧域迁到医学文献场景
- 验证 `subdomain` 是否已能作为前置语境参与抽取
- 查看当前输出的三元组是否已经具备进入后续 Grounding / Validation 的基础

---

## 2. 实验设置

### 输入

- 输入目录：`data/cleaned_papers`
- 样本数：12 篇论文

### 运行方式

```bash
python main.py \
  --data-dir data/cleaned_papers \
  --extraction-only \
  --extraction-report reports/extraction_claims.csv
```

### 输出

- 结果文件：`reports/extraction_claims.csv`

字段包括：

- `paper_id`
- `paper_title`
- `subdomain`
- `parent_domain`
- `claim_type`
- `subject`
- `relation`
- `object`
- `confidence`
- `section_title`
- `evidence`

## 5. 当前主要问题

这是当前汇报中最需要明确说清楚的部分。

### 5.1 过抽取

当前总数 2682 条，对 12 篇论文来说明显偏高。  
这说明模型把很多“论文中的可读叙述”也当成了“应该入图的知识”。

### 5.2 关系词过于自由

当前出现了不少不适合直接进入稳定 schema 的关系，例如：

- `candidate_susceptibility_gene_for`
- `may_contribute_to`
- `analyzed_in_study`
- `minimized`
- `demonstrated`
- `used_for`
- `used_in`
- `has_property`

启发：

- 当前 prompt 虽然要求使用控制关系词，但约束还不够强
- 需要把可接受关系进一步限制到更小的集合

### 5.3 句子型实体与叙事型实体仍然存在

当前统计中：

- 长实体条数：71
- 元叙事实体条数：131
- 可视为自由关系的条数：171

典型例子包括：

- `retrospective observational study with 76 patients`
- `patient in the ISKS cohort with the CHEK2 germline variant p.(Glu64Lys)`
- `presented case`
- `study`
- `we`
- `participants`

这些内容更像：

- 论文叙事
- study design
- 方法或写作信息

而不是适合长期入图的 biomedical knowledge node。

### 5.4 Methods / Introduction 抽取得太多

大量三元组来自：

- `Introduction`
- `Materials and Methods`

这些 section 中包含很多：

- 背景介绍
- 方法描述
- 写作性叙述

因此会带来更多松散三元组。

---

## 6. 当前结论

当前 Extraction 的状态可以概括为：

> 已经具备医学文献语境下的基本抽取能力，但还处于“可运行、可分析、不可直接入库”的阶段。

也就是说：

- 这一步已经不是空白状态
- 但当前还不能直接推进到完整主流程入库
- 在进入 Grounding / Validation / Evolution 之前，需要再收紧一轮 extraction

---

## 7. 下一步最优先改动建议

### 建议 1：收紧关系词集合

优先保留类似：

- `subclass_of`
- `type`
- `associated_with`
- `biomarker_for`
- `risk_factor_for`
- `has_diagnosis`
- `received_treatment`
- `measured_in`
- `inhibits`
- `activates`

对自由关系词进行压缩或拒绝。

### 建议 2：加强后过滤

优先过滤掉：

- `study`
- `we`
- `participants`
- `trial`
- `presented case`
- 长句型 subject/object

### 建议 3：降低 Introduction / Methods 权重

后续可考虑：

- 更偏向 `Abstract / Results / Discussion / Conclusions`
- 对 `Introduction / Methods` 更严格过滤

---

## 8. 当前阶段判断

当前最合理的推进方式不是直接入库，而是：

1. 先基于这轮结果继续收紧 Extraction
2. 再重新跑一次 `--extraction-only`
3. 确认总量、关系词和实体质量下降到更合理范围
4. 再进入完整主流程实验

一句话总结：

> 当前 Extraction 已经能抽出有价值的 biomedical triples，但还存在明显过抽取和 schema 漂移问题，需要再做一轮收紧后再进入入库主线。
